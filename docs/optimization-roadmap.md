# 创业客户管理工作台优化路线

更新时间：2026-06-06

## 当前判断

这个 App 已经具备个人接单管理的主干：客户、订单、收款、文件、导入备份、搜索和本地文件库。接下来最值得优化的方向不是继续堆大功能，而是减少每天重复操作的点击次数，并让“文件夹状态、订单状态、收款统计”这些容易混乱的地方更明确。

## 本轮已落地

- 顶部全局搜索支持 `Ctrl+K` / `Command+K` 聚焦，和界面里的快捷键提示一致。
- 工作台首页新增“新建客户”快捷按钮，适合先录客户、再补订单的接单流程。
- 订单页新增“待设计 / 待发货 / 待收款 / 逾期”快捷筛选，减少每天处理订单时反复打开下拉菜单的次数。
- 新建或编辑订单保存后会自动选中该订单并进入订单详情，方便马上上传文件、记录收款或补快递。

## 优先级 1：每天都会用的效率优化

1. 首页快捷操作区
   - 保留“新建客户”“新建订单”。
   - 后续可增加“打开客户文件库”“导入客户”“查看待收款”。
   - 目标：打开 App 后 1 次点击进入高频动作。

2. 订单列表快速筛选
   - 已增加“逾期”“待收款”“待发货”“待设计”快捷筛选按钮。
   - 后续可增加“今日到期”“三天内到期”。
   - 目标：不用下拉菜单就能处理当天最紧急的订单。

3. 保存后的去向优化
   - 已实现新建订单后自动选中新订单并打开右侧详情。
   - 新建客户后可提示“继续新建订单”。
   - 目标：录入后马上进入下一步工作。

## 优先级 2：降低出错概率

1. 文件库同步状态提示
   - 点击“刷新”后显示同步结果，例如隐藏了几个已删除文件夹、发现几个失败订单文件夹。
   - 目标：你知道刷新到底做了什么。

2. 收款输入保护
   - 输入收款金额超过未收金额时提示确认。
   - 目标：避免重复记录或多记收款。

3. 删除确认增强
   - 删除客户/订单时提示会影响哪些文件夹和统计。
   - 目标：删除前更安心。

## 优先级 3：后续扩展

1. 客户复购提醒
   - 根据上次下单时间标记“很久没联系”的老客户。

2. 订单模板
   - 常用项目如海报、名片、淘宝主图保存为模板，自动带出常用价格、尺寸和备注。

3. 云端只读查询
   - 沿用现有 `CloudReadModelExporter`，优先做手机/网页只读查询，不做编辑同步。
